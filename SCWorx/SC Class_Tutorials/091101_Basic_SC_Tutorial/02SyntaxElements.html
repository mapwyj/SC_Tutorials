// 9. Grouping Elements

  

  // Parentheses ( ) are used to: 

  

  // group expressions in order to specify order of execution:

  

  1 + (2 * 3) 

   

  // enclose arguments that are accompanying messages: 

  

  2.pow(3) 

  

  // create numerical arrays from "Matlab type" series notation: 

  

  (1..5) 

  

  // create Events from keyword-value pairs: 

 

  (freq: 440, amp: 0.1);


  // Events are special types of Dictionaries.

  e = (freq: 440, amp: 0.1);

  e.at('freq');

  e.at(\amp);

  

  // Brackets [] are used in two different ways, both associated with Collections:

  

  

  // A. Creating Arrays or other types of collections

  

  [1, 2, 5] // create an Array with the elements 1, 2, 5

  List[1, 2, 5] // create a List with the elements 1, 2, 5

  Set[1, 2, 5] // create a Set wit the elements 1, 2, 5 

   

  // B. Accessing the elements of collections for reading or writing of values: 

  

// Case 1: Arrays

  a = [1, 2, 5]; // Store an Array with elements 1, 2, 5 in variable a. 

  a[0]; // Access the first element of the Array stored in a. 

  a[0] = 'one' // store the symbol 'one' in the first element of Array a.

   // Case 1: Dictionaries

  a = Dictionary.new; // create a Collection of Class Dictionary and store it in variable a. 

  // the dictionary is empty. The post window shows: Dictionary[ ]. 

  a[\freq]; // try to access in the dictionary the value stored in key 'freq'.

  // since the dictionary is empty, nothing is stored in 'freq', therefore 

  // the result is nil.

    a[\freq] = 400; // store the number 400 in the dictionary under key 'freq'

  a[\freq] // access the value stored under key 'freq'. This is now 400. 

  a['freq'] * 2;

  a[\freq] = a[\freq] * 4; // store in freq the previous contents of freq multiplied by 4. 

a[\transpose] = 2; // store 2 at key 'transpose'

a[\freq] * a[\transpose];  // multiply the value stored in 'freq' by the value stored in 'transpose'


// Braces {} are used to:


// Define functions: 

{ arg a, b; a + b } // create (define) a function;

f = { arg a, b; a + b } // create a function and store it in f;

f.value(10, 5); // run the function stored in a with arguments 10 and 5;

// Define Classes. This is done in class definition files that need to be compiled with Command-K. 

// We will deal with Class definitions in later chapters. 

Nil { /* class definition code */ } 

// Define methods. This is done inside Class definitions. 

isNil { ^true }

// Delimiters

//The dot . is used in the following senses: 

// To attach a message to the receiving object that it is sent to: 

123.squared 

// To separate the decimal part of a floating point number from the integer part:

12.3


// To append an adverb to a binary operator. (Adverbs are identifiers or Integers that modify the behavior of an operator).

// Triple dots ... are used as ellipsis to collect multiple arguments into an array, in argument definitions: 

{ | ... args |  // or in multiple variable assignments:

#a, b ... rest = [1, pi, 10, true, inf];

 

// Double dots are used in notation of arithmetic series: 

(1..5)

(0, 0.1 .. 10)

// Comma is used to separate arguments:

   

  f.value(pi, 400)

   

  // elements of Collections:

   

    [pi, \a, 5]

     

  // or variables or arguments in declaration statements: 

   

    { arg a, b; 

   

  // or in multiple assignment statements: 

   

    #freq, amp = #[440, 0.1]

   

// Semicolon is used to separate statements. The last statement of a program (function) does not need to end in a semicolon.

a.postln; 

b = a.squared

// Pipe signs | | are used to delimit an argument declaration statement: 

{ | a, b | a + b } // This is alternative notation to: 

arg a, b;
