SEDataChan


a channel of data, usually contained in an SEData object.

SEDataChan can contain anything and makes; for different kinds of data

(such as numerical only), SEDataChan has several subclasses.



a = SEDataChan(\max, [1,2,3,\freq, nil]);

a.dump;


a.at(1);

a[0];

a[[3,1]];


a.copyRange(1, 3);


// for data which consist entirely of numbers, use

a = SENumDataChan(\max, [1,2,3]); 


// for efficiency, use RawArrays where possible. 

// e.g. use Signal (or DoubleArray if needed) for floats, 

a = SENumDataChan(\max, Signal[1,2,3]);

a = SENumDataChan(\max, DoubleArray[1,2,3]);


// and use Int32Array (or Int16Array if range is safe) for integers

a = SENumDataChan(\max, Int16Array[1,2,3]); 

a = SENumDataChan(\max, Int32Array[1,2,3]);



// for categorical data: 

a = SECatDataChan(\max, [1,2,3,\freq, 2, 3, \freq]);

a.repertoire; // all values that occur


// for time series, soundfiles are the best storage format:

a = SETimeSeriesChan.fromSF("sounds/a11wlk01.wav", \apollo);


a.analyse;


// after analysis, a datchan can suggest a spec so one can scale 

// it into [0, 1] range: 

a.suggestSpec.postcs;

a.makeSpec(max: 1.0, step: 0.01, default: 0.0); // make that a.spec and adjust e.g. max, step, default 

a.spec.postcs; // this is the spec a has now.

a.dump;
