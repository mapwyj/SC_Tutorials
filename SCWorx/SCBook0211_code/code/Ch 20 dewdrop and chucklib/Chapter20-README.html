This directory contains the extension libraries described in Chapter 20, along with code examples illustrating their use.



Directory layout


code-examples/ -- Includes the examples from the chapter text, plus a couple of other examples that were too long for the printed book.


dewdrop_lib/ -- The entire dewdrop_lib and chucklib libraries, as of revision 438 in the Quarks subversion repository.


addToQuarksDIRECTORY/ -- Descriptor (.quark) files for the dewdrop_lib quarks, to be used for (optional) quark-based installation.


MathLib/ -- A separate Quark, written by Julian Rohrhuber and others. Included here because chucklib uses some of its classes.


noodling-example/ -- A small and simple, but complete, algorithmic composition using chucklib.



Installation


The easiest way to install is to copy the dewdrop_lib and MathLib directories straight into your user extension directory (or system extension directory, if they should be available to all users on the machine). If you aren't sure of these locations, use the following commands:


Platform.userExtensionDir; // for user extension directory

Platform.systemExtensionDir; // for system extension directory


After copying the directories, recompile the class library. Chucklib prints the following message on start-up, so you can check for it to confirm successful installation.


Loaded 13 chucklib files from [your extension directory]/dewdrop_lib/ddwChucklib/Prototypes/.


IMPORTANT: ddwGUIEnhancementsForSwingOSC includes some extensions to the Java GUI classes from the SwingOSC library. If you do not have SwingOSC, but you install my enhancements for SwingOSC, the library will fail to compile because there are extensions for nonexistent classes. If you get such a warning, remove this directory (and only this directory) from your extension directory.


If you are using SwingOSC, make sure to install this folder because some of the GUI features of dewdrop_lib will not work without it.



If you like, you may use the Quarks package manager to simplify installation and uninstallation of individual library components. The following instructions put the Quark files in the right place. You do not need to go through these steps if you copied the library folders directly into the extensions directory as above.


1. In your user application support directory, create a directory called quarks/ if it doesn't exist already. (Note, the application support directory is different from the extension directory. Use the command Platform.userAppSupportDir to identify the full path.)


2. Copy the entire dewdrop_lib directory into this location. That is, you should have quarks/dewdrop_lib/ddwChucklib, quarks/dewdrop_lib/ddwCommon etc.


3. Also in the quarks/ directory, create a directory called DIRECTORY (it must be capitalized). (If you are already using quarks, this directory will already exist -- in that case, do not overwrite the existing directory.)


4. Copy the *.quark files from addToQuarksDIRECTORY into the DIRECTORY folder. (Do not copy the addToQuarksDIRECTORY itself, only the files within it.)


5. Recompile the library, and execute Quarks.gui. You should see the ddw* quarks in the list.



Code examples


In most cases, it's assumed the server will be running before you start to execute example code.


The examples in the code-examples folder include ample comments and should be straightforward to run.


The piece in noodling-example/ illustrates a structured code design for a piece that can be performed live. The logic is quite bit more complicated than the other examples. Usage notes are in the noodle-readme.html file.
